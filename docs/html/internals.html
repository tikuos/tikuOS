Kernel Internals

This page covers the low-level implementation details of the TikuOS kernel: how protothreads work at the macro level, how the event queue is structured, how the timer process scans for expirations, and how atomic sections provide interrupt safety.

Protothread Mechanics

TikuOS protothreads use Duff's device — a C technique where a switch statement jumps to a case label embedded inside the function body. The protothread state is a single unsigned short (lc_t) that stores the __LINE__ number where execution should resume.

Local Continuation Primitives

typedef unsigned short lc_t;

#define LC_INIT(s)    s = 0
#define LC_RESUME(s)  switch(s) { case 0:
#define LC_SET(s)     do { (s) = __LINE__; case __LINE__:; } while(0)
#define LC_END(s)     }

LC_SET stores the current line number and creates a case label at the same position. When LC_RESUME is called on the next invocation, the switch jumps directly to that label.

PT_BEGIN

Opens the switch, sets PT_YIELD_FLAG = 1, and calls LC_RESUME:

#define PT_BEGIN(pt) {             \
  char PT_YIELD_FLAG = 1;          \
  if(PT_YIELD_FLAG) {;}            \
  LC_RESUME((pt)->lc)

The PT_YIELD_FLAG is set to 1 on every entry. The if(PT_YIELD_FLAG) {;} suppresses unused-variable warnings. When the function is called for the first time, lc is 0 so the switch falls through to case 0: and execution starts from the top.

PT_YIELD

Saves the current position, then checks the yield flag to decide whether to return or continue:

#define PT_YIELD(pt)             \
  do {                           \
    PT_YIELD_FLAG = 0;           \
    LC_SET((pt)->lc);            \
    if(PT_YIELD_FLAG == 0) {     \
      return PT_YIELDED;         \
    }                            \
  } while(0)

On the first pass through this code, PT_YIELD_FLAG is set to 0 and LC_SET saves __LINE__. Since the flag is 0, the function returns PT_YIELDED. On the next call, LC_RESUME jumps back to the case __LINE__: label inside LC_SET, but now PT_YIELD_FLAG is 1 (set by PT_BEGIN), so the if is false and execution continues past the yield point.

PT_YIELD_UNTIL

Same as PT_YIELD but also checks a condition:

#define PT_YIELD_UNTIL(pt, cond)               \
  do {                                         \
    PT_YIELD_FLAG = 0;                         \
    LC_SET((pt)->lc);                          \
    if((PT_YIELD_FLAG == 0) || !(cond)) {      \
      return PT_YIELDED;                       \
    }                                          \
  } while(0)

Returns PT_YIELDED if either: (a) this is the first pass (flag is 0), or (b) this is a resume but the condition is still false.

PT_END

Closes the switch, resets the protothread, and returns PT_ENDED:

#define PT_END(pt)                  \
  LC_END((pt)->lc);                 \
  PT_YIELD_FLAG = 0;                \
  PT_INIT(pt);                      \
  return PT_ENDED;                  \
}

How Resume Works

Consider a simple example:

TIKU_PROCESS_THREAD(my_proc, ev, data)
{
    TIKU_PROCESS_BEGIN();       /* line 10: switch(lc) { case 0: */
    do_something();
    TIKU_PROCESS_YIELD();       /* line 12: case 12: */
    do_more();
    TIKU_PROCESS_END();         /* closes switch, returns PT_ENDED */
}

First call: lc = 0, switch jumps to case 0:, do_something() runs, YIELD saves lc = 12 and returns. Second call: lc = 12, switch jumps to case 12:, do_more() runs, PT_END returns PT_ENDED.

Warning: Local (stack) variables are not preserved across yield points because the function returns and its stack frame is destroyed. Use static variables or process-local storage for data that must persist across yields.

Event Queue Implementation

Structure

struct event_item {
    tiku_event_data_t data;      /* void * payload */
    struct tiku_process *p;      /* target process (NULL = broadcast) */
    tiku_event_t ev;             /* event type (uint8_t) */
};

static struct event_item queue[TIKU_QUEUE_SIZE];  /* 32 entries */
static volatile uint8_t q_head = 0;
static volatile uint8_t q_len = 0;

Circular Buffer

Events are enqueued at index (q_head + q_len) % TIKU_QUEUE_SIZE and dequeued from q_head. The queue size is 32 (a power of 2 for fast modulo). Both q_head and q_len are volatile uint8_t because ISRs may call tiku_process_post().

Atomic Access

Both tiku_process_post() and tiku_process_run() wrap their queue operations in tiku_atomic_enter() / tiku_atomic_exit(). This ensures that an ISR posting an event cannot corrupt the queue state while the main loop is dequeuing.

Broadcast

When the target process is TIKU_PROCESS_BROADCAST (NULL), the dispatcher iterates the process linked list. The next-pointer is pre-saved before calling into the process, so that if the process modifies the list (e.g., by exiting), the iteration remains safe:

struct tiku_process *p, *next;
for (p = tiku_process_list_head; p != NULL; p = next) {
    next = p->next;       /* pre-save before dispatch */
    call_process(p, ev, data);
}

Timer Process

A single process (tiku_timer_process) manages all software timers. It maintains a singly-linked list of active timers and scans for expirations when polled.

timer_is_due()

Uses wraparound-safe unsigned arithmetic:

static inline int timer_is_due(struct tiku_timer *t,
                               tiku_clock_time_t now) {
    return (tiku_clock_time_t)(now - t->start) >= t->interval;
}

The subtraction now - t->start is performed in unsigned arithmetic, so it handles clock wraparound correctly. Even if now has wrapped past zero, the elapsed time calculation remains valid as long as the interval is less than half the tiku_clock_time_t range.

Callback Dispatch

Callbacks are dispatched in the context of the process that created the timer, using TIKU_PROCESS_CONTEXT_BEGIN / CONTEXT_END:

if (t->mode == TIKU_TIMER_MODE_CALLBACK && t->func != NULL) {
    TIKU_PROCESS_CONTEXT_BEGIN(t->p);
    t->func(t->ptr);
    TIKU_PROCESS_CONTEXT_END(t->p);
}

This saves and restores tiku_current_process so that TIKU_THIS() returns the correct process inside the callback, even though the callback physically runs inside the timer process thread.

EXITED Cleanup

When the timer process receives a TIKU_EVENT_EXITED broadcast, it scans the timer list and removes all timers belonging to the dead process:

if (ev == TIKU_EVENT_EXITED) {
    struct tiku_process *dead = data;
    struct tiku_timer **pp = &timer_list;
    while (*pp != NULL) {
        if ((*pp)->p == dead) {
            struct tiku_timer *victim = *pp;
            *pp = victim->next;
            victim->active = 0;
        } else {
            pp = &(*pp)->next;
        }
    }
}

Without this cleanup, timers for dead processes would fire and attempt to post events to processes that no longer exist.

Rescan Pattern

After dispatching an expired timer, the scan restarts from the beginning (goto rescan) because the callback or event handler might modify the timer list (by setting, stopping, or resetting timers).

Atomic Sections

TikuOS atomic sections are nesting-aware. They save and restore the GIE (Global Interrupt Enable) bit only on the outermost pair:

tiku_atomic_enter();     /* saves SR, disables GIE */
    tiku_atomic_enter(); /* increments nesting counter */
    tiku_atomic_exit();  /* decrements counter, does NOT restore GIE */
tiku_atomic_exit();      /* outermost: restores original SR (with GIE) */

This design is critical for ISR safety. If an ISR calls code that uses atomic sections, the nested enter/exit will see a nesting count greater than zero and will not prematurely re-enable interrupts while still inside the ISR.

Note: On MSP430, the atomic enter saves the Status Register (SR) which contains the GIE bit, then clears GIE. The atomic exit restores the saved SR, which atomically re-enables interrupts if they were enabled before.

Boot Sequence

The deterministic boot sequence in main.c and boot/tiku_boot.c:

  1. Watchdog off — tiku_watchdog_off() halts the watchdog immediately to prevent unwanted reset during initialization.
  2. CPU init — tiku_boot_init_cpu() configures the DCO frequency, sets GPIO to a known state, and clears LPM5 lock.
  3. Memory init — tiku_boot_init_memory() calls tiku_mem_init() which initializes the region registry, activates MPU NVM protection, and performs arch-specific memory setup.
  4. Peripherals — tiku_boot_init_peripherals() initializes UART (so printf works) and the system clock.
  5. Services — tiku_boot_init_services() calls tiku_sched_init() which initializes the process system, hardware timer, and software timers.
  6. Tests — if TEST_ENABLE is set, test_run_all() runs the test suite.
  7. Scheduler loop — tiku_sched_loop() autostarts processes, enables interrupts, and enters the main event loop.
int main(void)
{
    tiku_watchdog_off();
    tiku_cpu_full_init(MAIN_CPU_FREQ);

#if TEST_ENABLE
    test_run_all();
#endif

    tiku_sched_loop();   /* never returns */
    return 0;
}
Key invariant: Global interrupts are NOT enabled until tiku_sched_loop() calls __enable_interrupt(). All initialization runs with interrupts disabled. This was a deliberate design choice after discovering bugs where premature GIE in hardware timer init and boot caused timer ticks to fire before the process system was ready to handle them.

Bugs Discovered During Testing

Three significant bugs were found by TikuBench automated testing and informed the current design:

Premature GIE in htimer init

The hardware timer initialization originally called __enable_interrupt() as part of starting the timer peripheral. This caused Timer_A ISRs to fire before the process system and software timer subsystem were initialized, leading to crashes when the ISR tried to poll a process that did not yet exist. The fix was to defer GIE to tiku_sched_loop().

Premature GIE in boot

A similar issue in the boot sequence where UART initialization enabled interrupts. An incoming UART byte would trigger an ISR that posted an event to a process that had not been started, corrupting the uninitialized event queue.

Missing EXITED broadcast

When a process exited, the kernel did not broadcast TIKU_EVENT_EXITED to other processes. The timer process never learned about dead processes and continued to hold references to their timers. When those timers expired, the timer process would attempt to post events to freed process structures, causing undefined behavior. The fix was to add an EXITED broadcast in tiku_process_exit() and a cleanup handler in the timer process.
