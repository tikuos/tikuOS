Timers

TikuOS provides two tiers of timers: software timers for process-level scheduling (millisecond resolution) and hardware timers for precise, interrupt-level timing (microsecond resolution).

Two Timer Modes

Feature Software Timers Hardware Timers
Resolution ~7.8 ms (1/128 s) ~1 us (1 MHz)
Tick constant TIKU_CLOCK_SECOND (128) TIKU_HTIMER_SECOND (~1,000,000)
Callback context Timer process (cooperative) ISR (interrupt level)
Delivery Event or callback Callback only
Struct struct tiku_timer struct tiku_htimer

Software Timer Struct

struct tiku_timer {
    struct tiku_timer   *next;       /* linked list (managed by kernel) */
    tiku_clock_time_t    interval;   /* ticks between firings           */
    tiku_clock_time_t    start;      /* tick count when timer was set   */
    struct tiku_process *process;    /* target process (event mode)     */
    void (*callback)(void *);        /* callback function (cb mode)     */
    void                *cb_data;    /* data passed to callback         */
    uint8_t              active;     /* 1 if armed, 0 if stopped        */
};

You never need to fill these fields manually — the API functions handle initialization.

Event Timers

An event timer delivers TIKU_EVENT_TIMER to a target process when it expires. This is the most common timer pattern in TikuOS.

Setting an event timer

static struct tiku_timer my_timer;

/* Fire after 2 seconds, deliver event to current process */
tiku_timer_set_event(&my_timer, TIKU_CLOCK_SECOND * 2,
                     &my_process);

Handling the event

TIKU_PROCESS_THREAD(my_process, ev, data)
{
    static struct tiku_timer timer;

    TIKU_PROCESS_BEGIN();

    tiku_timer_set_event(&timer, TIKU_CLOCK_SECOND,
                         &my_process);

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_TIMER);

        /* Timer fired -- do periodic work */
        toggle_led();

        /* Reschedule (drift-free) */
        tiku_timer_reset(&timer);
    }

    TIKU_PROCESS_END();
}

Callback Timers

A callback timer invokes a function when it expires. The callback runs in the timer process context (not in an ISR), so it is safe to call most kernel functions from it.

Setting a callback timer

static struct tiku_timer cb_timer;

void on_timeout(void *data)
{
    uint8_t *flag = (uint8_t *)data;
    *flag = 1;
}

static uint8_t timeout_flag = 0;

tiku_timer_set_callback(&cb_timer, TIKU_CLOCK_SECOND * 10,
                        on_timeout, &timeout_flag);

Callback timer example: debounce

static struct tiku_timer debounce_timer;

void debounce_done(void *data)
{
    /* 50 ms debounce period elapsed -- read button state */
    if (button_pressed()) {
        tiku_process_post(&app_process, EV_BUTTON_PRESS, NULL);
    }
}

/* Called from button ISR handler process */
tiku_timer_set_callback(&debounce_timer,
                        TIKU_CLOCK_SECOND / 20,   /* ~50 ms */
                        debounce_done, NULL);

Rescheduling

tiku_timer_reset(&timer) — drift-free

Reschedules the timer relative to its previous expiration time. If the timer was supposed to fire at tick 256 with an interval of 128, the next expiration is tick 384 — regardless of when the process actually handled the event. This prevents cumulative timing drift.

/* Drift-free 1 Hz periodic timer */
tiku_timer_reset(&timer);

tiku_timer_restart(&timer) — from now

Reschedules the timer relative to the current time. Useful for timeout patterns where you want "N seconds from right now," not periodic firing:

/* Reset inactivity timeout: 30 seconds from now */
tiku_timer_restart(&inactivity_timer);

Stopping a Timer

tiku_timer_stop(&my_timer);

Removes the timer from the active list. The timer struct can be reused immediately with a new set_event or set_callback call.

Timer Queries

Function Returns Description
tiku_timer_expired(&t) uint8_t Non-zero if the timer has fired and not been reset.
tiku_timer_remaining(&t) tiku_clock_time_t Ticks remaining until expiration (0 if expired).
tiku_timer_expiration_time(&t) tiku_clock_time_t Absolute tick count when the timer will fire.
tiku_timer_any_pending() uint8_t Non-zero if any software timer is currently active.
tiku_timer_next_expiration() tiku_clock_time_t Tick count of the earliest pending timer expiration. 
/* Check if we should enter low-power mode */
if (!tiku_timer_any_pending()) {
    enter_lpm4();   /* deepest sleep, no timers running */
} else {
    tiku_clock_time_t next = tiku_timer_next_expiration();
    configure_wakeup(next);
    enter_lpm3();
}

Hardware Timers

Hardware timers (struct tiku_htimer) provide microsecond-resolution timing by programming the MCU's Timer_A peripheral directly. Callbacks run in interrupt context.

Hardware timer struct

struct tiku_htimer {
    tiku_htimer_time_t   time;       /* expiration time (htimer ticks)  */
    void (*callback)(struct tiku_htimer *ht, void *data);
    void                *data;       /* user data for callback          */
};

Setting a hardware timer

static struct tiku_htimer ht;

void precise_callback(struct tiku_htimer *ht, void *data)
{
    /* Runs in ISR context -- keep it short! */
    TIKU_BOARD_LED1_OUT ^= TIKU_BOARD_LED1_PIN;
}

/* Fire 500 us from now */
tiku_htimer_time_t now = TIKU_HTIMER_NOW();
tiku_htimer_set(&ht, now + TIKU_HTIMER_SECOND / 2000,
                precise_callback, NULL);

Hardware timer macros

Macro Description
TIKU_HTIMER_NOW() Returns the current hardware timer tick count.
TIKU_HTIMER_TIME(ht) Returns the scheduled expiration time of hardware timer ht.
TIKU_HTIMER_SECOND Number of hardware timer ticks per second (~1,000,000 at 1 MHz).

Guard time

Guard time: When setting a hardware timer, the expiration time must be far enough in the future for the hardware to latch the compare value before the counter passes it. If you schedule a timer that is too close to TIKU_HTIMER_NOW() (within a few ticks), the timer may not fire until the counter wraps around. Always add a small margin: 
/* Safe: at least 10 ticks of margin */
tiku_htimer_set(&ht, TIKU_HTIMER_NOW() + delay + 10,
                callback, NULL);

Periodic hardware timer

Hardware timers are one-shot. To create a periodic hardware timer, reschedule from within the callback:

#define PERIOD (TIKU_HTIMER_SECOND / 1000)  /* 1 ms */

void periodic_isr(struct tiku_htimer *ht, void *data)
{
    /* Do time-critical work */
    sample_adc();

    /* Reschedule relative to current expiration (drift-free) */
    tiku_htimer_set(ht, TIKU_HTIMER_TIME(ht) + PERIOD,
                    periodic_isr, data);
}
ISR context: Hardware timer callbacks run inside an interrupt handler. Keep them as short as possible. Do not call tiku_process_post() or any blocking function. Use tiku_process_poll() to wake a process if you need to do heavier work after the ISR.

Timer Constants

Constant Value Description
TIKU_CLOCK_SECOND 128 Software timer ticks per second. Driven by the 32.768 kHz ACLK crystal divided down. One tick is approximately 7.8 ms.
TIKU_HTIMER_SECOND ~1,000,000 Hardware timer ticks per second. Driven by SMCLK at 1 MHz (or the configured DCO frequency). One tick is approximately 1 us.

Common time conversions

/* Software timer durations */
TIKU_CLOCK_SECOND          /* 1 second      */
TIKU_CLOCK_SECOND / 2      /* 500 ms        */
TIKU_CLOCK_SECOND / 4      /* 250 ms        */
TIKU_CLOCK_SECOND / 10     /* ~100 ms       */
TIKU_CLOCK_SECOND * 5      /* 5 seconds     */
TIKU_CLOCK_SECOND * 60     /* 1 minute      */

/* Hardware timer durations */
TIKU_HTIMER_SECOND              /* 1 second  */
TIKU_HTIMER_SECOND / 1000      /* 1 ms      */
TIKU_HTIMER_SECOND / 1000000   /* 1 us      */

Choosing the Right Timer

Use Case Timer Type Why
Periodic LED blink Event timer Low precision is fine; process-level handling.
Inactivity timeout Callback timer Single action on expiry, no process needed.
Sensor sampling at 1 kHz Hardware timer Needs 1 ms precision; callback triggers ADC in ISR.
UART byte timeout Hardware timer Microsecond resolution for character gap detection.
Debouncing a button Callback timer 50 ms delay, callback checks pin state.
Watchdog-style heartbeat Event timer Process checks health on each timer event.
