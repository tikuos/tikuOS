Processes

Processes are the fundamental unit of execution in TikuOS. Each process is a protothread — a stackless, cooperative task that yields control back to the scheduler at explicit yield points.

What Is a Process?

A TikuOS process is a lightweight cooperative task built on the protothread mechanism. Unlike preemptive threads, protothreads:

  • Share a single stack (no per-thread stack allocation)
  • Yield only at explicit yield points (cooperative, not preemptive)
  • Use C's switch/case mechanism internally for continuation
  • Consume only 2 bytes of RAM for continuation state

This makes them ideal for ultra-low-memory microcontrollers like the MSP430 where 2 KB of SRAM is all you have.

Declaring a Process

TIKU_PROCESS(name, strname)

The standard way to declare a process. Creates a struct tiku_process and forward-declares the thread function:

TIKU_PROCESS(my_process, "My Process");

TIKU_PROCESS_WITH_LOCAL(name, strname, local_type)

Declares a process with an associated block of typed local storage. The local data is allocated statically alongside the process struct and accessible via process->local:

typedef struct {
    uint16_t counter;
    uint8_t  state;
} my_locals_t;

TIKU_PROCESS_WITH_LOCAL(my_process, "My Process", my_locals_t);

TIKU_PROCESS_TYPED(name, strname, type)

Declares a process with a custom process struct type. Use this when you need to extend the process struct with additional fields:

typedef struct {
    struct tiku_process p;
    uint16_t sensor_id;
} sensor_process_t;

TIKU_PROCESS_TYPED(sensor_process, "Sensor", sensor_process_t);

Writing the Thread Function

TIKU_PROCESS_THREAD(my_process, ev, data)
{
    static uint16_t count;    /* must be static! */

    TIKU_PROCESS_BEGIN();

    count = 0;

    while (1) {
        TIKU_PROCESS_WAIT_EVENT();
        count++;
    }

    TIKU_PROCESS_END();
}

The macro TIKU_PROCESS_THREAD(name, ev, data) expands to a static function with the signature:

static int name_thread(struct tiku_process *process,
                       tiku_event_t ev,
                       void *data)

Parameters ev and data carry the event that woke the process. The scheduler calls this function each time an event is dispatched to the process.

Process Lifecycle

State Description
INIT Process is started and receives TIKU_EVENT_INIT. Initialization code before the first yield runs here.
Running Process is actively executing code between yield points.
Yielded Process has yielded at a WAIT or YIELD macro. The scheduler will resume it when the next matching event arrives.
Exited Process has exited (via PROCESS_EXIT, reaching PROCESS_END, or being killed externally). It receives TIKU_EVENT_EXIT.

TIKU_PROCESS_BEGIN / END

Every thread function must be bracketed by these two macros:

TIKU_PROCESS_THREAD(name, ev, data)
{
    TIKU_PROCESS_BEGIN();
    /* ... process body ... */
    TIKU_PROCESS_END();
}

TIKU_PROCESS_BEGIN() expands to a switch statement that jumps to the saved continuation point. TIKU_PROCESS_END() closes the switch, marks the process as exited, and returns. Omitting either macro will cause a compile error.

Waiting and Yielding

TIKU_PROCESS_WAIT_EVENT()

Yields the protothread and resumes on the next event of any type:

TIKU_PROCESS_WAIT_EVENT();
/* resumes here on any event */

TIKU_PROCESS_WAIT_EVENT_UNTIL(condition)

Yields and resumes only when the given condition is true. Events that do not satisfy the condition are silently discarded (the thread yields again):

TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_TIMER);
/* only resumes when a TIMER event arrives */

TIKU_PROCESS_YIELD()

Yields once unconditionally and resumes on the next scheduler invocation:

TIKU_PROCESS_YIELD();
/* resumes on the very next event dispatch */

TIKU_PROCESS_YIELD_UNTIL(condition)

Yields repeatedly until the condition becomes true:

TIKU_PROCESS_YIELD_UNTIL(sensor_ready());
/* resumes once sensor_ready() returns true */

Starting Processes

Autostart (most common)

TIKU_AUTOSTART_PROCESSES(&proc_a, &proc_b, &proc_c);

Processes listed here are started automatically during boot. Each receives an TIKU_EVENT_INIT event.

Manual start from another process

tiku_process_start(&child_process);

Scheduler-level start

tiku_sched_start(&my_process);

Stopping Processes

From inside the process

TIKU_PROCESS_THREAD(my_process, ev, data)
{
    TIKU_PROCESS_BEGIN();

    if (error_condition) {
        TIKU_PROCESS_EXIT();   /* immediately exits */
    }

    TIKU_PROCESS_END();        /* natural exit */
}

From outside the process

tiku_process_exit(&target_process);

The target process receives TIKU_EVENT_EXIT and is removed from the scheduler's process list.

Natural end

If execution flows past the last statement and reaches TIKU_PROCESS_END(), the process exits naturally.

Important Rules

No switch statements inside threads. Because protothreads use switch/case internally for continuation, you cannot use switch statements inside the protothread body (between BEGIN and END). Use if/else if chains instead.
No local variables across yield points. Protothreads are stackless. When a protothread yields, its stack frame is destroyed. Any non-static local variable will be uninitialized when the thread resumes. Declare persistent variables as static or use process-local storage via TIKU_PROCESS_WITH_LOCAL. 
/* WRONG -- count is garbage after yield */
int count = 0;
while (1) {
    TIKU_PROCESS_WAIT_EVENT();
    count++;  /* undefined behavior! */
}

/* CORRECT -- static persists across yields */
static int count = 0;
while (1) {
    TIKU_PROCESS_WAIT_EVENT();
    count++;  /* works as expected */
}
Each yield must be on a unique source line. The continuation mechanism uses __LINE__ to create case labels. Two yield macros on the same source line will collide and cause a compile error. Place each WAIT, YIELD, or EXIT on its own line.

The Process Struct

struct tiku_process {
    struct tiku_process *next;     /* linked list pointer       */
    const char          *name;     /* human-readable label      */
    int (*thread)(struct tiku_process *, tiku_event_t, void *);
                                    /* protothread entry point   */
    struct pt            pt;       /* continuation state (2 B)  */
    uint8_t              is_running; /* 1 if active, 0 if exited */
    void                *local;    /* optional local storage    */
};

The next pointer chains all active processes into a singly-linked list managed by the scheduler. The pt field holds only an unsigned short (the line number of the last yield), making each process extremely lightweight.

Code Examples

One-shot process

TIKU_PROCESS(init_hw, "HW Init");

TIKU_PROCESS_THREAD(init_hw, ev, data)
{
    TIKU_PROCESS_BEGIN();

    /* Runs once and exits */
    configure_gpio();
    configure_uart();
    tiku_process_start(&app_process);

    TIKU_PROCESS_END();
}

Process with local storage

typedef struct {
    uint16_t sample_count;
    int16_t  last_reading;
} sensor_local_t;

TIKU_PROCESS_WITH_LOCAL(sensor_proc, "Sensor", sensor_local_t);

TIKU_PROCESS_THREAD(sensor_proc, ev, data)
{
    sensor_local_t *local = (sensor_local_t *)process->local;

    TIKU_PROCESS_BEGIN();

    local->sample_count = 0;

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_TIMER);
        local->last_reading = read_sensor();
        local->sample_count++;
    }

    TIKU_PROCESS_END();
}

Parent-child process pattern

TIKU_PROCESS(parent, "Parent");
TIKU_PROCESS(child, "Child");

TIKU_PROCESS_THREAD(parent, ev, data)
{
    TIKU_PROCESS_BEGIN();

    /* Start child and wait for it to finish */
    tiku_process_start(&child);
    TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_EXIT);

    /* Child is done, continue */
    process_results();

    TIKU_PROCESS_END();
}

TIKU_PROCESS_THREAD(child, ev, data)
{
    TIKU_PROCESS_BEGIN();

    do_work();
    /* Falls through to PROCESS_END and exits naturally */

    TIKU_PROCESS_END();
}
