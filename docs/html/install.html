Installation

Set up the complete TikuOS development environment: compiler, flash tool, serial utilities, and hardware.

MSP430-GCC Toolchain

TikuOS uses the TI-provided MSP430-GCC open-source compiler. Download it from the TI MSP430-GCC page and install to ~/tigcc.

Linux

# Download and extract the toolchain
wget https://dr-download.ti.com/software-development/ide-configuration-compiler-or-debugger/MD-LlCjWuAbzH/9.3.1.2/msp430-gcc-full-linux-x64-installer-9.3.1.2.7z
7z x msp430-gcc-full-linux-x64-installer-9.3.1.2.7z
chmod +x msp430-gcc-full-linux-x64-installer-9.3.1.2.run
./msp430-gcc-full-linux-x64-installer-9.3.1.2.run --prefix ~/tigcc

# Add to your PATH
echo 'export PATH="$HOME/tigcc/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

macOS

# Download the macOS installer from TI's website and run:
./msp430-gcc-full-osx-installer-9.3.1.2.app --prefix ~/tigcc

# Add to your PATH
echo 'export PATH="$HOME/tigcc/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

mspdebug

mspdebug is the open-source tool used to flash firmware onto MSP430 devices and provide GDB-compatible debugging.

Linux (apt)

sudo apt update
sudo apt install mspdebug

macOS (Homebrew)

brew install mspdebug

From source

git clone https://github.com/dlbeer/mspdebug.git
cd mspdebug
make
sudo make install
Note: On Linux, mspdebug requires libmsp430.so (the TI library) for the tilib driver used by eZ-FET debuggers. This library is included with the MSP430-GCC toolchain in ~/tigcc/bin/. Make sure it is on your LD_LIBRARY_PATH: 
export LD_LIBRARY_PATH="$HOME/tigcc/bin:$LD_LIBRARY_PATH"

Python 3 and pyserial

Python is needed for TikuBench (the automated test harness) and for the serial monitor helper script.

# Install Python 3 (if not already present)
sudo apt install python3 python3-pip   # Linux
brew install python3                    # macOS

# Install pyserial
pip3 install pyserial

Hardware

TikuOS targets the MSP430FR5969 LaunchPad (MSP-EXP430FR5969). This board includes:

  • MSP430FR5969 microcontroller (16-bit RISC, 8 MHz, 2 KB SRAM, 64 KB FRAM)
  • On-board eZ-FET debugger/programmer (USB)
  • Backchannel UART over the same USB cable
  • Two user LEDs (P1.0 red, P4.6 green) and two buttons (P1.1, P4.5)

Connect the LaunchPad to your computer via the USB cable. It should enumerate as both a debug interface and a serial port.

Toolchain Verification

Run these commands to confirm everything is installed correctly:

# Verify the compiler
msp430-elf-gcc --version
# Expected: msp430-elf-gcc (Mitto Systems Limited - msp430-gcc 9.3.1.2) ...

# Verify mspdebug
mspdebug --version
# Expected: MSPDebug version 0.25 ...

# Verify Python and pyserial
python3 -c "import serial; print(serial.VERSION)"
# Expected: 3.5 (or similar)

# Check that the LaunchPad serial port is visible
ls /dev/ttyACM*
# Expected: /dev/ttyACM0 (or /dev/ttyACM1)

Common Issues

Permission denied on /dev/ttyACM0

On Linux, serial ports are owned by the dialout group. Add your user to this group and log out/in:

sudo usermod -aG dialout $USER
# Log out and back in for the change to take effect

tilib vs ezfet driver

The MSP430FR5969 LaunchPad uses the eZ-FET on-board debugger. When using mspdebug, you must specify the tilib driver:

mspdebug tilib "prog TikuOS.out"

If you see an error about libmsp430.so not found, ensure LD_LIBRARY_PATH includes the directory containing that library (typically ~/tigcc/bin).

Known issue: Some eZ-FET firmware versions have a bug where sending 0x00 over the backchannel UART triggers an unexpected reset of the debug interface. If you experience connection drops, try updating the eZ-FET firmware via TI's Code Composer Studio or avoid transmitting null bytes over the backchannel.

Device not found

If mspdebug cannot find the device, check that the USB cable is connected and the LaunchPad power LED is on. On Linux, verify with:

lsusb | grep "Texas Instruments"
# Should show: Texas Instruments ... MSP Tools Driver

Multiple LaunchPads

If multiple MSP430 boards are connected, mspdebug may pick the wrong one. Use the --usb-list option to enumerate devices and --usb-index to select a specific board.
