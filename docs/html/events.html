Events & IPC

TikuOS processes communicate through events — lightweight (type, data) pairs delivered via a circular queue — and through typed channels for structured inter-process data transfer.

Event Model

An event in TikuOS is a pair: an integer event type and an optional void * data pointer. Events are stored in a global circular queue with 32 entries. The scheduler drains this queue on each iteration of the main loop, dispatching events to their target processes.

/* Internal event queue entry */
struct tiku_event {
    tiku_event_t         type;     /* event type identifier     */
    void                *data;     /* optional payload pointer  */
    struct tiku_process *target;   /* destination process       */
};

System Events

Event Value Description
TIKU_EVENT_INIT 0x80 Delivered once when a process is started. Use for one-time initialization.
TIKU_EVENT_EXIT 0x81 Delivered when a process is about to be removed from the scheduler.
TIKU_EVENT_CONTINUE 0x82 Generic wake-up event with no specific meaning. Used internally.
TIKU_EVENT_POLL 0x83 Delivered when tiku_process_poll() is called. Signals that the process should check for new data or state changes.
TIKU_EVENT_TIMER 0x84 Delivered when a software event timer expires.
TIKU_EVENT_USER 0x90 Base value for application-defined events.

Custom Events

Define application-specific events starting from TIKU_EVENT_USER:

#define EV_SENSOR_READY   (TIKU_EVENT_USER + 0)
#define EV_DATA_LOGGED    (TIKU_EVENT_USER + 1)
#define EV_BUTTON_PRESS   (TIKU_EVENT_USER + 2)
#define EV_ALARM          (TIKU_EVENT_USER + 3)

Posting Events

Targeted post

Send an event to a specific process:

/* Post a sensor-ready event with a data pointer */
tiku_process_post(&logger_process, EV_SENSOR_READY, &reading);

The event is enqueued and delivered on the next scheduler iteration. The data pointer is passed through to the target process's thread function.

Broadcasting

Send an event to all running processes:

tiku_process_post(TIKU_PROCESS_BROADCAST, EV_ALARM, NULL);

The scheduler delivers the event to every active process in the process list.

Polling

Request that a process be woken up with a TIKU_EVENT_POLL:

tiku_process_poll(&uart_process);
ISR-safe: tiku_process_poll() is safe to call from an interrupt service routine (ISR). It sets a flag on the target process and does not touch the event queue directly. The scheduler checks poll flags at the top of each loop iteration and delivers TIKU_EVENT_POLL events before draining the regular queue. This is the recommended way to signal a process from an ISR.
ISR safety for post: tiku_process_post() is not safe to call from an ISR because it modifies the event queue without disabling interrupts. From ISR context, always use tiku_process_poll() instead.

Code example: event-driven sensor pipeline

/* Sensor process: reads sensor and notifies logger */
TIKU_PROCESS_THREAD(sensor_proc, ev, data)
{
    static struct tiku_timer timer;
    static int16_t reading;

    TIKU_PROCESS_BEGIN();

    tiku_timer_set_event(&timer, TIKU_CLOCK_SECOND * 5,
                         &sensor_proc);

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_TIMER);
        reading = read_temperature();
        tiku_process_post(&logger_proc, EV_SENSOR_READY,
                          &reading);
        tiku_timer_reset(&timer);
    }

    TIKU_PROCESS_END();
}

/* Logger process: waits for sensor data */
TIKU_PROCESS_THREAD(logger_proc, ev, data)
{
    TIKU_PROCESS_BEGIN();

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == EV_SENSOR_READY);
        int16_t *reading = (int16_t *)data;
        log_to_fram(*reading);
    }

    TIKU_PROCESS_END();
}

Event Queue Queries

These functions inspect the state of the global event queue:

Function Returns Description
tiku_event_queue_empty() uint8_t Non-zero if the queue has no pending events.
tiku_event_queue_full() uint8_t Non-zero if the queue has no room for more events.
tiku_event_queue_length() uint8_t Number of events currently in the queue (0..32).
tiku_event_queue_space() uint8_t Number of free slots remaining in the queue. 
if (tiku_event_queue_full()) {
    /* Cannot post -- queue is full, handle gracefully */
    error_count++;
}

Channels

Channels provide typed, bounded, FIFO data transfer between processes. Unlike raw events (which carry only a void *), channels transfer copies of typed data through a statically-allocated ring buffer.

Declaring a Channel

/* Declare a channel named "temp_ch" carrying int16_t with depth 8 */
TIKU_CHANNEL_DECLARE(temp_ch, int16_t, 8);

This macro expands to a struct containing a ring buffer of 8 int16_t entries, plus head/tail indices and a count.

Initializing

temp_ch_init();

Resets the channel to empty. Call this once during process initialization.

Putting Data

int16_t sample = 2350;
temp_ch_put(&sample);

Copies the value into the channel buffer. Returns non-zero if the channel was full (data not written).

Getting Data

int16_t value;
temp_ch_get(&value);

Copies the oldest entry out of the channel. Returns non-zero if the channel was empty (no data read).

Query Functions

Function Description
tiku_channel_is_empty(&ch) Returns non-zero if the channel has no data.
tiku_channel_free(&ch) Returns the number of free slots in the channel.

Producer-Consumer Pattern

/* Shared channel: 16 entries of sensor_reading_t */
typedef struct {
    int16_t  temperature;
    uint16_t timestamp;
} sensor_reading_t;

TIKU_CHANNEL_DECLARE(readings_ch, sensor_reading_t, 16);

/* --- Producer process --- */
TIKU_PROCESS(producer, "Producer");

TIKU_PROCESS_THREAD(producer, ev, data)
{
    static struct tiku_timer timer;
    static sensor_reading_t sample;

    TIKU_PROCESS_BEGIN();

    readings_ch_init();
    tiku_timer_set_event(&timer, TIKU_CLOCK_SECOND,
                         &producer);

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == TIKU_EVENT_TIMER);

        sample.temperature = read_sensor();
        sample.timestamp   = tiku_clock_time();

        if (readings_ch_put(&sample) == 0) {
            /* Notify consumer that data is available */
            tiku_process_post(&consumer, EV_SENSOR_READY,
                              NULL);
        }

        tiku_timer_reset(&timer);
    }

    TIKU_PROCESS_END();
}

/* --- Consumer process --- */
TIKU_PROCESS(consumer, "Consumer");

TIKU_PROCESS_THREAD(consumer, ev, data)
{
    static sensor_reading_t entry;

    TIKU_PROCESS_BEGIN();

    while (1) {
        TIKU_PROCESS_WAIT_EVENT_UNTIL(ev == EV_SENSOR_READY);

        /* Drain all available readings */
        while (readings_ch_get(&entry) == 0) {
            log_reading(entry.temperature, entry.timestamp);
        }
    }

    TIKU_PROCESS_END();
}
Thread safety: Channel operations are not interrupt-safe by default. If an ISR produces data, use tiku_process_poll() from the ISR to wake the consumer, and perform the actual _put() from process context (e.g., in a POLL event handler that copies data from a volatile ISR buffer into the channel).
