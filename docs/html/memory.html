Memory Management

TikuOS uses zero dynamic allocation. All memory is statically allocated and managed through purpose-built allocators: an arena for batch-lifetime buffers, a pool for fixed-size objects, and a persistent FRAM store for data that survives reboot. An MPU layer protects NVM from stray writes.

Design Principles

  • No malloc/free — malloc() on small SRAM (2-8 KB) leads to fragmentation that is fatal on microcontrollers with no MMU.
  • All static — every buffer is a caller-provided static array. Allocators manage the buffer but do not own or allocate it.
  • Platform-aware — the region registry maps the physical memory layout at boot, and allocators validate that their buffers are in the correct memory type (SRAM for arenas, NVM for persistent storage).

tiku_mem_init()

Entry point for the memory subsystem. Called during boot from tiku_boot_init_memory(). Orchestrates initialization in order:

  1. Region registry — must be available first so arena and persist registrations can validate their buffers.
  2. MPU init — activates NVM write-protection before anything else runs.
  3. Arch init — platform-specific memory hardware setup via the HAL.
void tiku_mem_init(void)
{
    tiku_mem_arch_size_t count;
    tiku_region_init(tiku_region_arch_get_table(&count), count);
    tiku_mpu_init();
    tiku_mem_arch_init();
}

Arena Allocator

A bump-pointer allocator for groups of allocations that share a lifetime. O(1) alloc, O(1) bulk free, zero per-object metadata, zero fragmentation.

How It Works

Each allocation advances an offset forward through the backing buffer. There is no individual free — all allocations are discarded at once by resetting the offset to zero.

/* Typical arena usage */
static uint8_t buf[256];
static tiku_arena_t arena;

tiku_arena_create(&arena, buf, sizeof(buf), 1);

void *a = tiku_arena_alloc(&arena, 32);  /* O(1) */
void *b = tiku_arena_alloc(&arena, 64);  /* O(1) */

tiku_arena_reset(&arena);                /* O(1), reclaims all */

API

Function Description
tiku_arena_create(arena, buf, size, id) Initialize an arena over a caller-provided buffer. Validates the buffer resides in SRAM via the region registry.
tiku_arena_alloc(arena, size) Bump-pointer allocation. Returns an aligned pointer or NULL if full. Tracks peak high-water mark.
tiku_arena_reset(arena) O(1) bulk free. Sets offset to zero. Does not zero memory. Peak is preserved across resets.
tiku_arena_secure_reset(arena) Same as reset but first overwrites the entire buffer with zeros via a volatile pointer. O(n). Use for cryptographic keys or credentials.
tiku_arena_stats(arena, stats) Fills a tiku_mem_stats_t with total_bytes, used_bytes, peak_bytes, alloc_count.

When to Use

Arenas are ideal for temporary scratch buffers during packet processing, sensor data batching, or any scenario where a group of allocations is used and discarded together.

Pool Allocator

A fixed-size block allocator with an embedded freelist. O(1) alloc, O(1) free, zero per-block metadata overhead, zero fragmentation.

How It Works

The buffer is divided into equal-sized blocks chained into an embedded freelist. Each free block stores a next-pointer inside its own memory. When allocated, the entire block is available to the caller. When freed, the first sizeof(void *) bytes are repurposed as the freelist pointer.

/* Typical pool usage */
static uint8_t buf[640];
static tiku_pool_t pool;

tiku_pool_create(&pool, buf, 32, 20, 2);  /* 20 blocks of 32 bytes */

void *blk = tiku_pool_alloc(&pool);       /* O(1) */
/* ... use blk ... */
tiku_pool_free(&pool, blk);               /* O(1) */

API

Function Description
tiku_pool_create(pool, buf, block_size, block_count, id) Divide buffer into blocks, chain into freelist. Block size rounded up to alignment and clamped to min sizeof(void *).
tiku_pool_alloc(pool) Pop the freelist head. Returns NULL if pool is empty. Tracks used_count and peak_count.
tiku_pool_free(pool, ptr) Push block back onto freelist. Validates ptr is within pool range and aligned to a block boundary.
tiku_pool_reset(pool) Re-chain all blocks into freelist. O(n) in block_count. Peak preserved.
tiku_pool_stats(pool, stats) Snapshot of total, used, peak bytes and count.

TIKU_POOL_DEBUG Poisoning

When TIKU_POOL_DEBUG is non-zero, tiku_pool_free() writes 0xDE to all bytes of a freed block (after the freelist pointer). This makes use-after-free bugs immediately visible in memory dumps and often causes a crash rather than silent corruption. Disable in production to avoid the per-free overhead.

When to Use

Pools are ideal for repeated alloc/free of same-sized objects: message buffers, sensor readings, network packets, or any structure that is allocated and freed individually throughout the application's lifetime.

Persistent FRAM Store

A key-value store for non-volatile data that survives reboot. Entries are registered at boot with caller-provided NVM buffers. A magic number (TIKU_PERSIST_MAGIC = 0x544B5553) distinguishes valid entries from uninitialized NVM.

API

Function Description
tiku_persist_init(store) Scan all slots: entries with correct magic and valid flag are kept, others cleared. Call once at boot.
tiku_persist_register(store, key, fram_buf, capacity) Register an NVM buffer under a key. If the key already exists, updates the pointer but preserves existing data.
tiku_persist_read(store, key, buf, buf_size, out_len) Copy from NVM to a caller's SRAM buffer. NVM may have wait states on some platforms.
tiku_persist_write(store, key, data, data_len) Copy data into NVM buffer. Increments write_count for wear monitoring. Caller must handle MPU unlock.
tiku_persist_delete(store, key) Clear the entry slot with memset so the key can no longer be found.

Wear Monitoring

Each entry tracks a write_count. The function tiku_persist_wear_check() returns 1 if the count exceeds TIKU_PERSIST_WEAR_THRESHOLD (default: 1 billion cycles). FRAM endurance is typically 10^15 cycles, but tracking matters for safety-critical systems and hot keys that are written frequently.

Region Registry

Maps the platform's physical memory layout at boot time. Subsystems query it to verify that their buffers are in the correct memory type.

Region Types

Type Description
TIKU_MEM_REGION_SRAM Volatile SRAM (arenas live here)
TIKU_MEM_REGION_NVM Non-volatile memory — FRAM, EEPROM (persistent store lives here)
TIKU_MEM_REGION_PERIPHERAL Memory-mapped peripheral registers
TIKU_MEM_REGION_FLASH Read-only flash (code, constants)

Key Functions

Function Description
tiku_region_init(table, count) Store the platform's region table and validate no overlaps.
tiku_region_contains(ptr, size, type) Check if a range falls within a region of the expected type.
tiku_region_claim(ptr, size, owner_id) Register ownership of a range. Detects overlapping claims.
tiku_region_unclaim(ptr) Release a previously claimed range.
tiku_region_get_type(ptr, out_type) Look up the region type for an address.

MPU (Memory Protection Unit)

The MPU provides NVM write-protection. Default policy: all segments are read+execute, no write. This prevents stray pointers and runaway code from corrupting non-volatile memory.

API

Function Description
tiku_mpu_init() Set all three segments to read+execute (no write) and enable MPU.
tiku_mpu_unlock_nvm() Add write permission to all segments. Returns saved state for restoration.
tiku_mpu_lock_nvm(saved) Restore MPU state from a prior unlock.
tiku_mpu_scoped_write(fn, ctx) Disable interrupts, unlock NVM, call fn(ctx), relock, re-enable interrupts.
tiku_mpu_enable_violation_nmi() Switch MPU violation response from device reset to NMI.
tiku_mpu_get_violation_flags() Read per-segment violation flags (bits [2:0]).
tiku_mpu_clear_violation_flags() Reset all violation flags to zero.

Scoped Write Pattern

/* Manual unlock/lock */
uint16_t saved = tiku_mpu_unlock_nvm();
/* ... write to NVM ... */
tiku_mpu_lock_nvm(saved);

/* Or use the convenience wrapper (disables interrupts too) */
tiku_mpu_scoped_write(my_write_fn, &my_context);
Warning: Interrupts are disabled during scoped writes because an ISR that fires while NVM is unlocked could itself write to NVM through a bug, defeating the protection. Keep write functions short to avoid missing interrupts.

Error Codes

Shared by all allocator types in the memory module:

Code Value Meaning
TIKU_MEM_OK 0 Operation succeeded
TIKU_MEM_ERR_INVALID -1 Invalid argument (NULL pointer, bad alignment, buffer not in correct region)
TIKU_MEM_ERR_NOMEM -2 Out of memory (arena full, destination buffer too small)
TIKU_MEM_ERR_FULL -3 Store or claim table is full (no free slots)
TIKU_MEM_ERR_NOT_FOUND -4 Key not found in persistent store, or address not in any region
