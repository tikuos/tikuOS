Quick Start

Get TikuOS built, flashed, and running on your MSP430FR5969 LaunchPad in under 5 minutes.

Prerequisites

Before you begin, make sure you have the following installed on your system:

  • MSP430-GCC toolchain — the TI-provided GCC compiler for MSP430. Install it to ~/tigcc or update the Makefile paths accordingly.
  • mspdebug — open-source flash and debug tool for MSP430 devices. Available via apt (Linux) or brew (macOS).
  • Python 3 with pyserial — required if you plan to run TikuBench automated tests or use the serial monitor helper.
  • Hardware — an MSP430FR5969 LaunchPad (MSP-EXP430FR5969) connected via USB.

See the Installation page for detailed setup instructions.

5-Minute Getting Started

1. Clone the repository

git clone https://github.com/your-org/tikuOS.git
cd tikuOS

2. Build the firmware

make MCU=msp430fr5969

3. Flash the device

make flash

4. Open a serial terminal

make monitor

Or manually connect with any serial terminal at 9600 baud, 8N1:

screen /dev/ttyACM0 9600

5. Done

You should see TikuOS boot output on the serial console. The default firmware initializes all kernel subsystems and enters the scheduler loop.

Build Commands

Command Description
make MCU=msp430fr5969 Build firmware for the FR5969 target (tests/examples auto-detected)
make flash Flash the compiled .out binary onto the connected LaunchPad
make monitor Open a serial monitor on the eZ-FET backchannel UART
make clean Remove all build artifacts
make APP=cli MCU=msp430fr5969 Build with the CLI application (excludes tests and examples)
Note: Only one of APP=, tests, or examples can be active at a time. When APP= is specified, tests and examples are automatically excluded from the build.

What Happens at Boot

When TikuOS starts on the MSP430, it executes a deterministic boot sequence:

  1. Watchdog off — the watchdog timer is halted immediately to prevent an unwanted reset during initialization.
  2. CPU init — the CPU clock is configured (DCO, MCLK, SMCLK, ACLK), GPIO pins are set to a known state, and FRAM wait states are configured.
  3. sched_init — the scheduler initializes the process list, event queue, and all timer subsystems (etimer, ctimer, rtimer).
  4. Autostart processes — any processes registered with TIKU_AUTOSTART_PROCESSES are started and receive an INIT event.
  5. sched_loop — the main event loop runs forever, dispatching events to processes and entering low-power mode when idle.
/* Simplified boot flow in main.c */
int main(void)
{
    tiku_watchdog_stop();
    tiku_cpu_init();
    tiku_sched_init();
    tiku_autostart_start(autostart_processes);
    tiku_sched_loop();          /* never returns */
    return 0;
}

Verify It Works

After flashing, connect a serial terminal to the LaunchPad's backchannel UART:

  • Port: /dev/ttyACM0 (Linux) or /dev/cu.usbmodemXXXX (macOS)
  • Baud rate: 9600
  • Data bits: 8, Parity: None, Stop bits: 1

If the build includes the CLI application (make APP=cli), you will see a tikuos> prompt. Type help to list available commands.

If the build includes the default test configuration (TEST_ENABLE 1 in tiku.h), you will see test results printed to the console.

Tip: If you see no output, double-check that your serial terminal is connected to the correct port and baud rate. On Linux, you may need to add your user to the dialout group for permission to access /dev/ttyACM0.
